Images

HTML pages are in fact containers of different elements, which can be included in the page. External elements should be supported by the browsers in order to be rendered correctly - that is, the Media (or MIME) type of the file should be supported.

As for images, all modern browsers generally supports:

  • JPEG or JPG
  • GIF
  • PNG

There is also the possibility to use SVG files. Contrary to the formats listed above, SVG are vector images (i.e. computed mathematically), whereas the other are raster images (i.e. determined by pixels).

How to include an image

In order to include an image you need:

  1. A file with the image with the proper extension. The file can be anywhere on the internet, but it must be available through the http protocol (thus avoid Google drive or Dropbox files)
  2. The img tag with an src attribute with the reference to the image: . The src attribute works like the href attribute for links: the attribute can be an absolute URL or a relative path.

Attributes for the image

The img tag has many attributes, but some of them are considered deprecated. For instance, if the original image is not of the right size, in the past it was common to use the width="100" and height="100" attributes for the determining the pixels of each image. Nowadays, though, it is suggested to use CSS properties instead.

An important attribute to use for accessibility/usability is the alt attribute, which should be used to describe the image for screen readers.

Often, we also use the attribute title with a descriptive text that shows on mouseover.

Basic guidelines for using images

Some advices:

  • Keep in mind that big images are also heavy to download, so sometimes it is necessary to reduce the size of the image with an editor.
  • Be sure that there is no copyright on the image and that you respect the license under which the image can be used.
  • Generally, it is better to download an image instead of using its absolute URL
